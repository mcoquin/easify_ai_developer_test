Financial Administration

Financial management in the Easify platform occurs primarily at the unit level.

Each unit can have:

invoices

payments

variation costs

financial documents

Blocks cannot contain invoices.

This rule ensures that every financial transaction corresponds to a sellable property.

Invoice lifecycle

Invoices typically move through these states:

Status	Description
Draft	Invoice is being prepared
Pending Approval	Waiting for project manager approval
Approved	Ready to send
Sent	Delivered to customer
Paid	Payment received

Only approved invoices may be sent.

Historical note

Older versions of the platform briefly experimented with block-level invoicing, but this approach was abandoned because it caused accounting inconsistencies.

Some internal documents may still reference this concept.

The current system does not support block-level invoices.
