Project Structure

The Easify platform models construction developments using a hierarchical system.

A Project represents a full development. Examples include apartment buildings, residential neighborhoods, or mixed-use developments.

Projects are divided into Blocks.

Blocks represent physical subdivisions of a project. For example:

Building A

Building B

Tower West

Tower East

Each block contains Units.

Units represent sellable entities such as apartments, houses, or commercial spaces.

Example:

Project: Riverside Gardens

Block A

Unit A1

Unit A2

Unit A3

Block B

Unit B1

Unit B2

Important note

Financial administration always occurs on units, not on blocks.

Blocks exist only as structural containers.

Special cases

Some developments contain underground parking structures.

In those cases the system may include additional logical subdivisions.

Example:

Block A

Residential units

Parking units

Storage units

Parking spaces are technically considered units in the system.

Known confusion

Many new users assume invoices can exist on blocks.
This is incorrect.

Invoices always belong to units.
