Platform Evolution

The Easify financial system has evolved significantly.

Earlier versions allowed more flexible accounting models.

For example:

invoices linked to project phases

experimental block-level financial tracking

optional unit aggregation

These models caused confusion.

Most were removed.

Removed features

The following features were deprecated:

Block-level invoicing
Phase-based invoice generation
Supplier cost aggregation at block level

These approaches made it difficult to trace costs to individual buyers.

Current rule

All invoices must be attached to a unit.
