Project Structure

The Easify platform represents construction developments using a hierarchical model.

A Project represents a complete development.

Examples:

apartment complex

residential neighborhood

mixed-use development

Projects are divided into Blocks.

Blocks represent physical structures such as:

Building A

Building B

Tower West

Tower East

Blocks contain Units.

Units represent sellable entities such as:

apartments

houses

parking spaces

storage rooms

commercial units

Example hierarchy:

Project: Riverside Residence

Block A
Unit A1
Unit A2
Unit A3

Block B
Unit B1
Unit B2

Financial responsibility

Units are considered the primary financial entity in the system.

Most financial records are attached to units.

Note for administrators

While most financial data belongs to units, internal accounting reports may aggregate data at the block level.

These block reports do not represent real invoices.

Historical confusion

Some legacy documentation incorrectly suggests blocks can contain invoices.

This was briefly tested during development but is no longer supported.
